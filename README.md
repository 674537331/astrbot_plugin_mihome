# 🌟 AstrBot MiHome Control

基于 [AstrBot](https://github.com/Soulter/AstrBot) 框架开发的支持自然语言调用的米家智能家居云端控制插件。

## ✨ 核心特性

* 🔒 **隔离式扫码登录**：登录流程通过独立子进程完成，避免阻塞主事件循环；认证凭证安全保存在 `plugin_data` 目录中，便于备份、迁移与卸载清理。
* 🚀 **场景直接调用**：直接通过米家APP内已有场景对多个设备进行复杂调整，支持指令和LLM工具自然语言调用！
* 🧠 **设备画像驱动 (Device Profiles)**：将用户别名、中文控制词与底层属性键解耦。插件内置常见设备的专属画像，可将“睡眠”“制冷”“童锁”等中文指令映射为对应设备可识别的底层属性和值。
* 📊 **中文状态展示**：基于本地画像与云端返回结果，对设备状态项进行中文翻译与单位美化展示。
* 🛡️ **柔性降级与防呆设计**：当设备暂时离线或状态读取失败时，插件会优先返回本地设备画像与引导信息，尽可能保证“可看、可控、可排障”。
* 🔑 **全设备兼容**：对所有设备均已适配直接开关，并支持通过原始属性（如 `temperature 26`）进行透传控制。

## 📖 使用指南

<details>
<summary><b>🚀 快速上手：从零到控制你的第一台设备 (点击展开)</b></summary>

对于初次使用的用户，请严格按照以下 5 步完成设备绑定与控制：

### 步骤 1：扫码授权登录
对机器人发送 `/米家登录`。机器人会返回一个授权链接，打开后使用米家 APP 扫码完成授权。

### 步骤 2：获取设备 DID（关键）
发送 `/刷新米家`。机器人会从云端拉取设备列表。记录下你要控制设备对应的 DID 数字。

### 步骤 3：配置设备别名
前往 AstrBot WebUI 插件配置页，找到 `device_map`，为 DID 配置一个别名：
```json
{
  "卧室净化器": "1234567",
  "客厅 空调": "8888888"
}
```

### 步骤 4：指定设备类别（推荐）
在 `device_category_map` 中填写别名对应的类别，以获得中文化支持。支持：`空调类别`、`净化器类别`、`风扇类别`、`蒸煮锅类别`、`空气炸锅类别`、`体脂秤类别`、`温湿度计类别`、`扫地机类别`。

### 步骤 5：开始测试
发送 `/米家详情 卧室净化器` 查看状态，或发送 `/米家控制 卧室净化器 开` 进行控制。

</details>

<details>
<summary><b>🎮 指令全集：管理、查询与控制 (点击展开)</b></summary>

### 🔑 账号与管理
* `/米家登录`：获取扫码授权链接。
* `/米家登出`：清除本地凭证。
* `/米家状态`：查看插件当前运行状态。

### 📡 设备查询
* `/刷新米家`：同步云端列表并刷新本地缓存。
* `/米家详情 [设备别名]`：查看设备实时状态（优先级：型号匹配 > 类别匹配 > 原始探测）。
* `/米家帮助 [设备别名]`：查看该设备支持的常用指令示例。

### 🎛️ 设备控制
* **基础控制**：`/米家控制 [设备别名] [开/关]`
* **高级属性**：`/米家控制 [设备别名] [属性] [值]`（支持中文映射或英文透传）
* **动作触发**：`/米家控制 [设备别名] [动作名]`（如：开始清扫、start_sweep）

</details>

<details>
<summary><b>🎬 米家智能场景支持（Scenes） (点击展开)</b></summary>

支持执行米家 APP 中已配置的“手动执行”场景。

### 📱 前置准备
在米家 APP “智能”页面创建一个**触发条件为“手动执行”**的场景，并命名（如：`晚安模式`）。

### 🎮 场景指令
* `/米家场景列表`：同步并查看所有可执行的云端场景及 ID。
* `/米家场景 [场景名或scene_id]`：立即执行指定场景。

**⚠️ 同名场景保护**：若存在多个同名场景，插件会拦截执行并提示改用唯一的 `scene_id`。

</details>

<details>
<summary><b>🤖 大模型自然语言联动（LLM Tool） (点击展开)</b></summary>

### 🛡️ 安全边界
为了安全考虑，大模型**仅允许调用“场景”能力**。不支持大模型直接调整设备属性（如改温度、开关机），以防幻觉误操作。

### ⚙️ 如何启用
1. 在插件配置页开启 `enable_scene_tool`。
2. 重载插件。
3. 手动执行一次 `/米家场景列表` 以刷新本地缓存。

### 💬 交互示例
* **你**：`@机器人 帮我执行晚安场景`
* **你**：`@机器人 帮我关下灯`（前提：米家 APP 有名为“关灯”的手动场景）

</details>

## 🔬 进阶与反馈

<details>
<summary><b>🧠 技术细节：三层降级解析机制 (点击展开)</b></summary>

本插件采用以下顺序解析设备能力：
1. **型号精准匹配**：若设备型号命中内置 `MODEL_PROFILES`，展现最完美的专属翻译。
2. **大类模板兜底**：若未命中型号但配置了类别，使用通用中文模板。
3. **原始能力探测**：若为 `无类别`，返回原始英文属性菜单，支持英文指令透传。

</details>

### 💡 设备画像未完全适配？
欢迎提交 Issue 帮助我们完善核心字典库（MODEL_PROFILES）。请附带 `/astrbot/data/plugin_data/astrbot_plugin_mihome` 下对应的 `.json` 文件，或在终端执行以下代码并提供输出：
```bash
python - <<'PY'
from mijiaAPI import get_device_info
import json
model = "你的设备model"
info = get_device_info(model)
print(json.dumps(info, ensure_ascii=False, indent=2))
PY
```

## ⚠️ 常见排障与常见问题

| 现象 | 可能原因与建议 |
| :--- | :--- |
| 设备找不到 / 共享异常 | 建议优先使用设备主账号扫码登录。 |
| 设备拒绝请求 | 可能修改了只读属性或值越界。请参考 `/米家帮助`。 |
| 详情页只有英文菜单 | 设备未内置且未配置类别。请检查 `device_category_map`。 |
| 提示实况获取失败 | 设备可能离线或处于深度休眠（如体脂秤未踩亮）。 |
| 网络异常 / 超时 | 可能发生 TLS 断连或被防火墙风控，建议稍后重试。 |
